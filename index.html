发一点感慨

今天建立这个个人主页对于我有着不一样的心情，和一定的特殊意义。至于是怎么样个意义，还没想好，慢慢讲，讲到哪算哪。昨晚略失眠，今天早上起床，不想好好学习，又没有堕落到打游戏的地步，于是想找点事情做。也许是因为知乎上不经意掠过的关于博客的一个帖子，也许是某年某月某一天羡慕过本科室友搞过的一个不能再简陋的独立博客。对写博客这件事慢慢植根在了脑海的深处，今天在一个百无聊赖的早上，终于完成了这个夙愿。

我想，搞一个自己的博客应该是每一个程序员最终的归宿吧(虽然从严格意义上说我不算程序员，但我以向这个身份靠近而光荣)。总有一天，你会在某种情境下做这件事。也许是想把自己积累的技术在网上记录下来，不只是在脑海中，在笔记本里总结。亦或是因为孤独，终日对着键盘与显示器，终日对着代码，终于，也想对着电脑说点人话，而不是if else了。（这么说当然不严谨，电脑还是会把人话变成成机器语言，看来想让写的东西来点文学性，还是不要那么认真。）于是便有了自己的博客。

今天本来没有那么大的兴致，非要搞成不可，因为觉得自己只是一个二手的程序员，之前只用matlab，c++的知识只停留在课本。只是因为做的东西用到c++，才从头学起，并以程序员自居，其实现在连半吊子都算不上。

已经凌晨三点半，好久没有这么发感慨，困的要死，今天就到这吧。

不要奢望某一天过上什么样的生活，你现在就可以过上想要的生活，那是不关乎物质条件的。可以富而贵，也可贫而清。任何时候都可以活的有尊严

ubuntu下 hexo+github page的配置遇到的问题

  1. 注册github
  2. 开始同名repository
  3. 本机git 和hexo配置

其他文章有讲，所以我主要记录一下遇到的问题

  1. hexo 的命令要先cd到其目录下进行
  2. 配置文件改写

deploy:
type: git
repo: https://github.com/yanyinhong/yanyinhong.github.io.git
branch: master

  1. hexo generate // hexo g，生成静态文件，会在当前目录下生成一个public文件夹
    hexo server // hexo s，启动本地服务
    hexo deploy // hexo d，部署到远程
    hexo d -g // 生成和部署
    hexo s -g // 生成和预览

    git clone https://github.com/wuchong/jacman.git themes/jacman
